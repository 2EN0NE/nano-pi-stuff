# @zenone/pi-recap

起草一行你上次在 Pi 会话中停在哪里的回顾——灵感来自 Claude Code 的会话回顾功能。

## 功能

- **按需使用：** 随时执行 `/recap` 获取 1-2 行摘要，了解之前在处理什么、完成了什么以及下一个待办步骤。在提交前、开 PR 前或长时间会话后非常有用。
- **返回时自动触发：** 当你回到一个已经空闲了一段时间的会话时，Pi 会自动起草回顾并在下一轮交互前显示——这样你就不需要回滚来重建上下文。

回顾由后台模型调用生成（通过宿主的 `completeSimple`，与 `smart-context` 相同的机制），使用当前会话的模型。这是尽力而为的：如果无法运行，它会静默跳过，绝不会阻塞你的交互轮次。

## 自动回顾触发条件

Pi 没有原生的终端焦点/失焦事件，因此与 Claude Code 的"失焦 ≥ 3 分钟"信号不同，此扩展使用自上次完成轮次以来的**挂钟空闲时间**：

1. 自上次 `agent_end` 以来至少过了 **3 分钟**。
2. 会话已有至少 **3 轮用户交互**。
3. **永远不会连续两次显示**——显示一次回顾后，下一次会等待新的活动。
4. 仅交互模式（`ctx.hasUI`）；在打印/无头模式下跳过。

实际效果（返回时等待你的回顾）是一样的；触发方式是基于时间而非基于焦点。

## 配置

自动回顾默认开启。使用环境变量禁用它：

```bash
export PI_ENABLE_AWAY_RECAP=0
```

无论此设置如何，`/recap` 始终可用。

## 安装

`.pi/settings.json`：

```json
{
  "packages": ["npm:@zenone/pi-recap"]
}
```
