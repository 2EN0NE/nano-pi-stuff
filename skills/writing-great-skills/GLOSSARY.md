# 词汇表——构建优秀技能

优秀技能的领域模型。一个技能的存在是为了从随机系统中提取确定性；根本美德是**可预测性（Predictability）**，下面的每个术语都是作用于它的杠杆。这是 [`writing-great-skills`](SKILL.md) 的已披露参考。

术语按轴分组：**调用（Invocation）**（技能如何被触达）、**信息层次（Information Hierarchy）**（内容如何组织）、**引导（Steering）**（agent 运行时行为如何塑造）和**修剪（Pruning）**（如何保持精益）。每个**失败模式**位于治愈它的杠杆旁边，标记为_失败模式_。

任意定义中的**粗体术语**本身在本词汇表中定义；通过其标题找到它们。

## 可预测性（Predictability）

技能使 agent 在每次运行时以相同的_方式_行为的程度——相同的过程，而不是相同的输出（头脑风暴技能应_可预测地_发散；其 token 变化，但其行为不变）。每个其他术语服务的根本美德——成本和可维护性是其症状，而非对手。

_避免_：一致性、可靠性、健壮性、输出确定性

## 调用（Invocation）

技能如何被触达——以及你为选择支付的两种负载。

### Model-Invoked

保留其 **description** 字段的技能，因此 agent 可以看到它并自主触发它——而且人类仍然可以输入其名称，因此 model-invocation 始终_包含_用户可触达。没有仅模型的状态：description 只会_添加_ agent 发现能力，从不删除人类的访问权限。每回合支付永久的 **context load** 以换取这种可发现性。可被其他技能调用，因为使其 agent 可发现的 description 也使其可被调用。当 model-invoked 技能的内容全是参考时，它也是共享参考的一个家：另一个技能可以调用它，因此多个技能需要的参考生活在一个地方。仅当 agent 必须自己触达该技能时选择 model-invocation；如果它从不手动触发，删除 description 并支付零 context load。

_避免_：能力、工具、能力

### User-Invoked

**description** 被剥离的技能——对 agent 不可见，仅通过人类输入其名称才可触达（仅用户，而 **model-invoked** 是用户_and_ agent）。用 agent 可发现性换取零 **context load**。由于它没有 description，除了人类没有东西可以触达它：没有其他技能可以调用它。

_避免_：过程、工作流、命令

### Description（描述）

技能的机器可读触发器，以及一个 **model-invoked** 技能被迫始终加载的 **context pointer**。它的存在_就是_调用轴：保留它，技能是 model-invoked（且可被其他技能触达）；删除它，技能是 **user-invoked**，仅通过人类可触达。model-invoked 技能的 **context load** 来源。

_避免_：前言、摘要

### Context Pointer（上下文指针）

在 agent 上下文中持有的引用，命名了某段上下文外的材料并对触达该材料的条件进行了编码。**description** 是顶级上下文指针（上下文窗口 → 技能）；指向已披露文件的指针是再向下一级的相同对象。其措辞，而非目标，决定 agent_何时_触达——以及_多可靠地_触达。弱措辞指针后的必要材料是一个方差 bug：先修复措辞，只有锐化失败时才内联材料。

_避免_：链接、引用、导入

### Context Load（上下文负载）

一个 **model-invoked** 技能强加给 agent 上下文窗口的成本——其 **description**，始终加载，花费 token 和注意力。**user-invoked** 技能通过没有 description 来逃避的成本，以及拆分为更多 model-invoked 技能的制动器。

_避免_：token 成本、上下文膨胀

### Cognitive Load（认知负载）

一个 **user-invoked** 技能强加给人类的成本——他们必须在脑中记住的内容：哪些技能存在以及何时使用每个（人类是索引）。**model-invocation** 通过具有 agent 可发现性来移除的成本，以及拆分为更多 user-invoked 技能的制动器。不是为了最小化的成本：它是人类自主性的代价，是一些技能保持 user-invoked 的原因。在人类判断重要时花费它；在不重要时移除它。

_避免_：人类索引、负担、开销

### Router Skill（路由器技能）

一个 **user-invoked** 技能，其工作是指向你的其他 user-invoked 技能——命名每个以及何时触达它——使人类只需记住一个技能而不是多个。它只能提示，不能触发它们：user-invoked 技能没有 **description**，所以除了人类没有东西可以触达它们。当 user-invoked 技能增多时的 **cognitive load** 治疗方法。

_避免_：调度器、菜单、注册表、索引、路由器过程

### Granularity（粒度）

你划分技能的精细程度。更细的划分花费两种负载之一：更多的 **model-invoked** 技能花费 **context load**（更多 description 拥挤在窗口中并争夺注意力）；更多的 **user-invoked** 技能花费 **cognitive load**（人类需要记忆和触达的更多）。两个切分指引划分：通过**调用方式**，当你有一个不同的**领先词**来触发它时拆分出一个 model-invoked 技能——一个你在提示中实际使用的触发词。通过**顺序**，当一系列 **steps** 中某个步骤的**后完成步骤**需要隐藏时拆分，因为将其隔离在自己的上下文中清除了后续。注意反向：合并序列会暴露各步骤的后完成步骤给后续，引发过早完成。

_避免_：块化、模块化

## 信息层次（Information Hierarchy）

技能内容如何组织，以及每个部分在梯子上有多深。

### 信息层次（Information Hierarchy）

技能内容按 agent 需要它的紧迫性排序——单个梯子，由两个切分产生：在文件中还是在指针后面，以及步骤还是参考。梯级：

- **Steps（步骤）**——文件中，主要的
- **Reference（参考）**，文件中——次要的
- **Reference（参考）**，已披露——在 **context pointer** 后面

没有 **steps** 的技能只使用底部两个梯级——通常是一个合法的扁平同级集（例如审查的每条规则在一个梯级上），这是一个很好的安排，不是坏味道。层次独立于调用方式：一个技能可以是 model- 或 user-invoked，无论它全是步骤、全是参考、或两者兼有。当技能有步骤时，应披露但未披露的文件内参考会埋没它们，并使关注它们变成抛硬币——一个方差杠杆，而不仅仅是可读性杠杆。保持梯子顶部清晰；将你能推送的都推下去。

_避免_：结构、组织、布局

### Steps（步骤）

agent 执行的有序动作——当技能有步骤时，其内容的主要层级，以及赢得其在 SKILL.md 中位置的部分。不是每个技能都有步骤：一个技能可以全是步骤（`tdd`）、全是 **reference**（审查）或两者兼有，独立于调用方式。每个步骤以**完成标准**结束，清晰或模糊。

_避免_：工作流、指令、编排

### Reference（参考）

agent 按需查阅的材料——定义、事实、参数、示例、条件性指令。当技能有 **steps** 时，它次于它们；当技能没有步骤时，它是全部内容；或者它完全存在于任何技能之外——见 **External Reference**。通过 **context pointers** 触达，是 **progressive disclosure** 的主要候选。

_避免_：支持材料、文档、背景

### External Reference（外部参考）

存在于技能系统之外的 **reference**——普通文件，没有 **description**，没有 **steps**，不可调用——任何技能可以指向它。不需要自己的调用的共享参考的家，也是两个 **user-invoked** 技能可以使用的唯一共享家，因为两者都没有 description 所以都不能调用另一个。

_避免_：文档、资源、知识库

### Progressive Disclosure（渐进式披露）

将 **reference** 向下移动梯子——从 SKILL.md 中移出，放在 **context pointer** 后面——使顶部保持清晰。主要不是 token 优化；它是**信息层次**如何被保护。由**分支**授权：披露仅某些分支需要的内容，内联每条路径都需要的内容，如果指针在必要材料上触发不可靠，锐化其措辞，只有锐化失败时才将其拉回内联。

_避免_：懒加载、分块

### Co-location（共置）

将 agent 需要一次使用的材料保持在一个地方——一个概念的定义、规则和注意事项在一个标题下，而不是分散在文件中——这样阅读一个部分时会带来其邻近部分。**信息层次**的文件内伴侣：层次对_内容多深_进行排序；共置决定_一旦放好，什么放在它旁边_。正确格式的参考正文没有公式；测试是一个技能应读起来像为 agent 编写的文档，分组的材料读起来就是那样，而分散的材料则不是。与**重复**不同：那是在两个位置重复一个含义，而分散是将单一含义碎片化到许多位置。

_避免_：分组、聚类、内聚

### Sprawl（蔓延）

_失败模式。_ 一个技能太长了——SKILL.md 的行数太多——无论它们是否过时或重复。即使是全部活跃、全部唯一的技能也可能蔓延。它伤害可读性（agent 在能行动之前要涉水更多，注意力在过量内容中变薄）、可维护性（每额外一行都是另一行需要保持**相关**）、和 token。治疗方法是**信息层次**：将 **reference** 向下推到 **context pointers** 后面，并按**分支**或顺序拆分，使每条路径只携带它需要的。与 **sediment**（因 stale 积累而长）和 **duplication**（因重复含义而长）不同——蔓延是长度本身，无论其原因。

_避免_：膨胀、长度、大小、冗长

## 引导（Steering）

塑造 agent 运行时行为朝向**可预测性**的杠杆。

### Branch（分支）

技能可以被调用的不同方式——技能处理的一个案例——因此不同的运行采用不同的技能路径。具有许多步骤的技能可能携带许多分支；线性的技能没有。

_避免_：路径、案例、分叉

### Leading Word（领先词）

一个紧凑的概念——也称为_主导词_——已经存在于模型的预训练中，agent 在运行技能时用它来思考。它通过调用模型已有的先验知识，在最少的 token 中编码行为原则（例如 _lesson_、_proximal zone of development_、_fog of war_、_tracer bullets_）。作为 token 重复，而不是作为句子，它在整个技能中累积分布式定义并锚定整个行为区域。自己创造也可以，只要你清晰定义它，但编造的词不招募先验知识——你支付的定 token 数相当于预训练词免费提供的量。先找现有词。

领先词为**可预测性**服务两次。在正文中它锚定**执行**——每次概念出现时 agent 触达相同的行为，在扁平参考内部，它将注意力集中在一类要寻找的事物上，每次运行招募正确的检查。在 **description** 中它锚定**调用**——而且不仅在技能内部：当相同的词存在于你的提示、文档和代码库中时，agent 将共享语言链接到技能并更可靠地触发它。用你实际想要技能时使用的领先词来措辞 description。

_避免_：关键词、术语、主题

### Completion Criterion（完成标准）

告诉 agent 一个工作单元已完成的条件——它判断的目标。两个属性使其成为杠杆，而不仅仅是质量。其**清晰度**（agent 能否区分已完成与未完成？）抵抗**过早完成**——模糊的边界（"已达成理解"）让 agent 声明已完成并滑到下一步；这个轴需要 _steps_ 才能起作用，因为过早完成是一个步骤间的失败。其**要求**（它要求多少）设置 **legwork**——"每个修改过的模型都已说明"迫使彻底的工作，而"生成变更列表"则不会——这个轴_不_绑定于步骤：它也可以约束扁平参考的整体，这就是为什么没有步骤的技能仍然带有详尽性标准（"每条规则都应用了"）。最强的标准既可检查又详尽。

_避免_：完成条件、退出条件、停止规则

### Legwork（背景工作）

agent 在单个步骤内部幕后做的工作——读取文件、探索代码库、进行更改、挖掘它需要的而不是转嫁给用户。它存在于步骤结构下方：从不用作自己的步骤，潜伏在措辞中，由 agent 而不是技能控制。**后完成步骤**的跨步骤拉力的步骤内对应物。由**领先词**（_comprehensive_、_thorough_）或要求工作详尽的**完成标准**提高——包括应用于扁平参考的需求轴，这才是驱动纯参考技能覆盖所有梯级的原因。当需求缺失或**过早完成**过早切断步骤时，它会变薄。

_避免_：范围、努力、勤勉、覆盖

### Post-Completion Steps（后完成步骤）

当前步骤之后的**步骤**。可见时，它们将 agent 向前拉入**过早完成**——看到越多，拉力越强；防御是通过将步骤序列拆分为两个来隐藏它们。

_避免_：视野、战争迷雾、前瞻

### Premature Completion（过早完成）

_失败模式。_ 在当前步骤真正完成之前结束它，因为 agent 的注意力滑到"已完成"而非工作上。一个步骤间失败：它需要**步骤**才能发生——没有步骤却提前退出的技能不是过早完成，而是在未满足的需求下的薄 **legwork**。两股力量之间的拔河：可见的**后完成步骤**（向前的拉力）和**完成标准**的清晰度（阻力——一个锐利、可检查的界限能承受；模糊的会退让）。模糊是必要条件：一个锐利的界限无论后面有多少步骤可见都抵抗拉力，因此从不匆忙的步骤不需要防御。两个杠杆可以控制匆忙的步骤，但要按顺序使用它们：**首先锐化界限**——它是本地且便宜的。只有标准不可减少地模糊_并且_你确实观察到匆忙时，才**隐藏后面的步骤**——而且隐藏只有在真正的上下文边界（user-invoked 的手动交接或子代理调度）时才有效；内联的 model-invoked 调用将后面的步骤留在上下文中，什么也清不掉。薄 legwork 的一个原因，但与其不同：legwork 即使在步骤完全完成时也可能很薄。

_避免_：过早闭合、匆忙、走捷径

### Negation（否定）

_失败模式。_ 通过禁止来引导——告诉 agent 不要做什么——将禁止的行为拖入上下文使其_更_可用，而不是更少。_不要想大象_，然后大象就是全部；_永远不要写冗长的注释_，然后冗长就是 agent 刚刚读到的模式。否定是一个弱修饰符，被强激活的概念淹没，因此禁令半读作执行该事的指令。其**领先词**是_大象_：禁止命名到框架中的任何东西。治疗：提示**正面**——描述目标行为（"写一行的注释"），使被禁止的从不被提及。禁令只在无法正面表述的硬性护栏上赢得其位置；即使如此，也要与正面目标配对，使注意力落在做什么上。

_避免_：讽刺反弹、不要提示、粉红色大象

## 修剪（Pruning）

保持技能精益——每个补救措施与它所治愈的失败配对。

### Single Source of Truth（单一真相来源）

每个含义恰好在一个权威位置的期望状态，因此技能行为的变化是一个位置的变化。**重复**是其违反。

_避免_：家、规范位置

### Duplication（重复）

_失败模式。_ 同一含义被赋予多个**单一真相来源**。它花费维护（改变一处，你必须改变其他处），花费 token，并将突出度推过其实际等级。**领先词**的意外对立面，后者通过重复一个 token（而不是含义）来故意提高注意力。

_避免_：重复、冗余

### Relevance（相关性）

一行是否仍然与技能的作用有关——用于决定保留什么的镜头。一行要么因为从未与任务相关（仅仅是说明，或应被披露的**分支**），要么因为过时而失去相关性：随着它所描述的行为或世界的变化而过时。较短的技能更容易保持相关，因为每行检查起来更便宜。与 **no-op** 不同：相关性问一行是否与任务有关，而不是它是否改变行为。

_避免_：承重、过时、新鲜度

### Sediment（沉积）

_失败模式。_ 旧内容的层次在技能中沉积，从未被清除，因为添加感觉安全而移除感觉有风险——因此过时和不相关的行积累起来，你必须深入挖掘才能找到仍在活跃的内容。任何没有修剪纪律的技能的默认命运；**相关性**的缓慢侵蚀，相对于**重复**的重复含义。

_避免_：积累、膨胀、积垢、腐烂

### No-Op（无操作）

_失败模式。_ 一条什么也改变不了的指令，因为模型默认已经做了——你支付负载告诉 agent 它本来就会做的事。测试：一行是否改变了与默认相比的行为？一行可以完全**相关**仍然是 no-op。使**领先词**免费的相同先验知识也使 no-op 毫无价值。

领先词是一种_技术_；No-Op 是对一行的_判定_——它们交叉。一个弱到无法击败默认的领先词是一个 no-op（当 agent 已经相当彻底时_be thorough_），修复是一个通过判定的更强的词（_relentless_），而不是不同的技术。因此 No-Op 测试——它是否改变了与默认相比的行为？——也是你评估领先词是否值得其重复的方式。这是模型相对而不是读者相对：两个人对一行是否是无操作存在分歧时，是在默认上存在分歧，并通过运行技能来解决，而不是通过辩论。

_避免_：冗余指令、重述显而易见的事、啰嗦
