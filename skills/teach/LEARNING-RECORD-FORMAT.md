# 学习记录格式

学习记录存放在 `./learning-records/` 中，使用顺序编号：`0001-slug.md`、`0002-slug.md`，依此类推。懒创建目录——仅当第一条记录写入时才创建。

它们是教学领域中的 ADR（架构决策记录）：捕获非显而易见的课程、关键见解和已明确的前置知识，这些将引导未来的学习。它们用于计算最近发展区。

## 模板

```md
# {已学内容或已确立知识的简短标题}

{1-3 句话：学到了什么（或确立了哪些前置知识），以及它为何对未来的学习很重要。}
```

以上就是完整的格式。一条学习记录可以只是一个段落。价值在于记录_已经知道_的内容和_为何_它改变了下一步教什么——而不是填写各个部分。

## 可选章节

仅在确实增加价值时才包含这些。大多数记录不需要它们。

- **状态** frontmatter（`active | superseded by LR-NNNN`）——当之前的理解被证明是错误的并被替换时有用。
- **证据**——用户如何展示了该理解（回答的问题、完成的练习、引用的先前经验）。当该主张可能被重新审视时有用。
- **影响**——这对未来的学习解锁了什么或排除了什么。当影响非显而易见时值得记录。

## 编号

扫描 `./learning-records/` 寻找现有的最大编号并加一。

## 何时写入学习记录

当以下任一情况成立时写入一条：

1. **用户展示了对复杂内容的真正理解**——不仅仅是接触过，而是有证据表明他们能正确使用该概念。这为下一步教什么设定了新的起点。
2. **用户公开了前置知识**——"我已经知道 X"。记录下来以免未来的学习重复教授。同时记录所声明的_深度_。
3. **误解被纠正**——用户之前相信了错误的东西，现在理解了原因。这些很有价值：它们能预测相关主题的未来障碍。
4. **任务因学习而转变**——用户发现自己关心的内容与最初所想的不同。交叉链接到 [[MISSION.md]] 并更新它。

### 什么_不_算

- 仅仅覆盖过的材料。覆盖不等于学习。等待证据。
- 已在 [[GLOSSARY.md]] 中简洁捕捉为术语定义的任何内容。不要重复。
- 逐次的活动日志。学习记录不是日记——它们是决策级的洞察。

## 取代

当后续记录与早期记录矛盾时（用户的理解加深或得到纠正），将旧记录标记为 `Status: superseded by LR-NNNN`，而不是删除它。理解如何演化的历史本身就是有用的信号。
