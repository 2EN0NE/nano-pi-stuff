# GLOSSARY.md 格式

`GLOSSARY.md` 是此教学工作区的规范性语言。所有解释性内容、练习和学习记录都应遵循其术语。构建词汇表本身就是学习的一部分：将一个概念压缩到精确定义中是用户理解该概念的证据。

## 结构

```md
# {主题} 词汇表

{一两句话描述此词汇表涵盖的主题。}

## 术语

**肌肥大 (Hypertrophy)**：
由重复训练中的机械张力和代谢压力驱动的肌肉生长。
_避免_：练大、增肌

**渐进超负荷 (Progressive overload)**：
随着时间系统地增加对肌肉的需求——通过负重、训练量或强度。
_避免_：加大力度、升级

**RPE（主观疲劳度）**：
一组动作感觉难易程度的 1-10 自评，其中 10 是力竭，8 意味还剩两次重复的余力。
_避免_：努力分数、强度评级
```

## 规则

- **仅在用户理解该术语时添加它。** 词汇表是压缩知识的记录，不是用户用来学习的词典。如果用户刚接触一个概念，等到他们能正确使用它时再将其升级到这里。
- **要有主见。** 当多个词表示同一概念时，选择最好的一个，并将其余列为要避免的别名。这就是语言如何压缩。
- **保持定义紧凑。** 一两句话。定义术语_是什么_，而不是它做什么或如何做。
- **在定义内部使用词汇表自身的术语。** 一旦一个术语进入词汇表，在任何地方都优先使用它——包括在其他定义内部。这就是让复杂术语之后更容易理解的方法。
- **当自然聚类出现时分组在子标题下**（例如 `## 解剖学`、`## 训练计划`）。当术语足够内聚时，扁平列表也可以。
- **明确标记歧义。** 如果某个术语在更广泛的领域中被随意使用，注明解决办法："在此工作区中，'set' 始终指工作组的动作——热身单独追踪。"
- **随着理解加深而修订。** 用户第一周写的定义可能在第六周就是错的。原地更新；不要留下过时的条目。
