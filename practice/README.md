## 分层的Prompt设计

Pi的Prompt机制涉及到用户级、项目级，替换Pi的原生Prompt或附加在尾部等策略可选。基于“没有银弹”原则，最佳实践应该是通过这一机制设计符合自己日常工作流的分层Prompt体系设计。举例如下：

| 文件                            | 主题                | 说明                                                                                                              |
| ------------------------------- | ------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `example-user-system-prompt.md` | 用户级别系统 Prompt | 放在`~/.pi/agent/AGENTS.md` ，作为无论什么类型Agent工作的共性要求，会附加在所有位置pi启动时pi代码原生prompt的后面 |
| `../AGENTS.md`                  | 项目级别系统Prompt  | 在该项目启动时附加在用户级别系统Prompt的后面                                                                      |
